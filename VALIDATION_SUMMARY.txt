        after response generation

QUESTION 3: When are tokens available?
ANSWER: ✅ After Claude generates response (barista runs post-generation)

QUESTION 4: Can hooks see output tokens?
ANSWER: ❌ No - hooks run BEFORE generation, only barista can see output

HOW IT WORKS
================================================================================

USER INTERACTION:
  User: "/escalate to opus"
           ↓
  HOOK (pre-response):
    - Detects escalate command
    - Estimates: 270 input tokens, 500 output tokens = 770 total
    - Records to DB with validation_id
    - Returns model routing (opus)
           ↓
  CLAUDE (processes):
    - Generates response with 450 actual output tokens
           ↓
  BARISTA (post-response):
    - Reads .context_window.current_usage
    - Sees: 268 input, 474 output = 742 actual tokens
    - POSTs to /api/validate with actual data
           ↓
  SERVICE:
    - Receives actual metrics
    - Calculates: token_error = (742 - 770) / 770 = -3.6%
    - Stores: validation result
           ↓
  DASHBOARD:
    - Shows: Estimated 770 vs Actual 742 (error: -3.6%) ✅
    - Accumulates: accuracy statistics

DEPLOYMENT (15 MINUTES)
================================================================================

Step 1: Create barista module (5 min)
   $ cat > ~/.claude/barista/modules/escalation-validation.sh << 'BARISTA'
   #!/bin/bash
   module_escalation_validation() {
     local input="$1"
     local total=$(echo "$input" | jq '.context_window.current_usage | ...')
     curl -s -X POST http://localhost:9000/api/validate \
       -H "Content-Type: application/json" \
       -d "{\"actual_total_tokens\": $total, ...}"
   }
   BARISTA
   $ chmod +x ~/.claude/barista/modules/escalation-validation.sh

Step 2: Enable module (2 min)
   $ echo 'MODULE_ESCALATION_VALIDATION="true"' >> ~/.claude/barista/barista.conf

Step 3: Rebuild service (3 min)
   $ cd /tmp/claude-escalate
   $ go build -o claude-escalate ./cmd/claude-escalate
   $ cp claude-escalate ~/.local/bin/escalation-manager

Step 4: Restart service (2 min)
   $ pkill -f "escalation-manager service"
   $ escalation-manager service --port 9000 &

Step 5: Verify setup (3 min)
   $ curl http://localhost:9000/api/validation/stats | jq .
   # Should show: {"total_metrics": 0, "validated": 0, ...}

Done! Validation system is now active.
